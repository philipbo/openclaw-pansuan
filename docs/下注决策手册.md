# 下注决策手册（版本 1）

> 目标：给自己一套「从不碰到下手」的完整流程，只保留长期有价值的下注场次。

---

## 一、先判定：这场「要不要碰」

### 1.1 必须直接跳过的场次（乱局）

满足任一条，默认本场为「乱局」，**不做价值下注，只看不买**：

1. **极端伤停局**
   - 单边 **≥5 人伤病** 且含多名主力（尤其门将 / 中卫 / 中锋）；或
   - 两队伤停人数差 **≥3 人**（例如 9 vs 2），且你不打算按伤停规则修正 λ。
2. **战意怪局**
   - 一方已夺冠 / 提前出线 / 已降级，战意高度不确定；
   - 关键杯赛 / 欧战前后，轮换规模大到阵容失真。
3. **末轮练兵 / 搅局**
   - 联赛最后几轮，尤其是最后一轮：练兵、送别、默契球风险高；
   - 多队积分博弈复杂（赢平负都能出线 / 保级）的局面。
4. **数据严重不足**
   - 主 / 客各自主场 / 客场样本 <5 场，或整体联赛场次 <8 场，泊松 λ 信噪比过低。

> 这些场次可以用来练习「看盘 / 看走势」，但**不纳入价值下注体系**。

## 二、价值扫描（泊松 + 欧指平均）

### 2.1 用泊松算出基础概率

1. 用主/客场 + 近 6 场 + 伤停修正算出 λ₁（主）、λ₂（客）。
2. 泊松 → 主 / 平 / 负概率：P主、P平、P客。
3. 泊松 → 总进球数分布：P(0 球…5+ 球)。

> 公式与 λ 细节见 `docs/泊松概率计算手册.md`。

### 2.2 欧指平均：胜平负 EV

1. 拿 **欧指平均** 主/平/客赔率 O主、O平、O客（初盘 / 即时 / 终盘）。
2. 计算期望值：
   - EV主 = P主 × O主
   - EV平 = P平 × O平
   - EV客 = P客 × O客
3. **只把 EV≥1.05 的方向标记为「价值候选」**，其余放弃。（EV 处于 1.00～1.05 的“边际”区间，默认记笔记但不强推）

> 若三边 EV 都 ≤1，则本场**不做胜平负价值下注**。

### 2.3 欧指平均：大小球 EV（可选）

1. 用总进球分布算出大 / 小某条线（如 2.5）概率：P大、P小。
2. 拿欧指平均大小球赔率 O大、O小，算：
   - EV大 = P大 × O大
   - EV小 = P小 × O小
3. 同样只保留 EV≥1.05 的一侧作为「大小球价值候选」。

> 没有明显 EV≥1.05 时，大小球更多作为「比分筛选参考」，不是主仓位。

## 三、盘口与走势过滤危险热门

### 3.1 平博：尖锐参考盘（胜平负 / 亚盘 / 大小球）

1. 记录平博自初盘到临场的 **盘+水 时间序列**：谁的赔率一路下、谁一路上。
2. 用你自己的公平赔率（1/P）对比当前平博赔率 O：
   - 若 O ≪ 1/P：价格被打得太低，是**危险热门区**，价值下注应回避；
   - 若 O ≫ 1/P：价格被打高，有机会是「反大众价值点」（需基本面确认）。

### 3.2 亚盘四家：澳彩 / 皇冠 / Bet365 / 易胜博

1. **看盘位**（初盘让球）：谁是庄家心中的正路上盘。
2. **看盘变**：升盘偏上、降盘偏下。
3. **看水位**：
   - 升盘 + 上盘低水 → 强势上盘盘（多数人也在博上盘）；
   - 降盘 + 上盘仍低水 → 常被理解为「诱下实挺上」，要防上盘热门坑。
4. **多家公司是否同向**：
   - 多家一起升盘+压低上盘水 → 一致强势盘；
   - 方向分裂 → 意图不清，尽量不靠盘感下注。

### 3.3 欧指客群盘：威廉 / Interwetten / 马会 / 中国竞彩网

1. 看某边赔率是否长期「比欧指平均更低」：
   - 本地盘一起压主胜 → 本地热门堆积区；
   - 本地盘抬高某边 → 本地冷门，在泊松支持下是逆向加分项。
2. 不把这些公司的赔率当真概率，只当**情绪与操盘意图的雷达**。

### 3.4 过滤规则

在第二步筛出的「价值候选」方向里，再用盘口过滤：

- 若某方向 **EV≥1.05**，但：
  - 平博 + 亚盘 + 本地盘都极端压低这一边的价格；
  - 你在足球本质上也只是「略倾向」而非明显优势；
    → 这类更可能是模型误差 / 大众热门，**宁愿放弃，不为 EV 一点点去冒灾难风险**。
- 优先保留：
  - **模型强烈支持（胜率高）**，
  - **欧指平均 EV≥1.05**，
  - **盘面没有全民一边倒的极端热门信号** 的方向。

## 四、让球盘：买上盘还是下盘

### 4.1 真实赢盘概率（用比分表聚合）

以主让半球（-0.5）为例：

- 上盘赢盘 = 主胜概率 P主；
- 下盘赢盘 = P平 + P客。

主让一球（-1）：

- 上盘赢盘 = 主队净胜 ≥2 球的比分概率之和；
- 上盘走水 = 主队净胜 1 球；
- 下盘赢盘 = 平 / 负比分概率之和。

> 不要靠盘感猜赢盘概率，用你已经算好的比分矩阵直接聚合。

### 4.2 水位 ≈ 赔率，算 EV

- 水位上 → O上 ≈ 1/水位上；
- 水位下 → O下 ≈ 1/水位下；
- EV上 = P上 × O上，EV下 = P下 × O下。

**只有 EV≥1.05 的一边，才是让球盘上的价值候选。**

### 4.3 盘感只做辅助，不做自动信号

- 不再用「盘深就下盘、盘浅就上盘」这种一刀切规则；
- 盘深 / 盘浅只帮助你识别：
  - 是否在做局（名气盘、末轮盘）；
  - 是否需要额外警惕热门方向；
- 真正决定买上盘还是下盘，**仍然要回到 EV≥1.05 的硬条件**。

## 五、比分玩法（若要参与）

1. 用泊松产生原始比分概率 P(i:j)。
2. 用风格 / 让球 / 大小球，对不同比分打简单权重 w(i:j)（见 `docs/比分建模与投注手册.md`）。
3. 用 S(i:j)=P(i:j)×w(i:j) 排序，选 2～4 个主攻比分；
4. 在有比分赔率时，也可以算 EV(比分)=P(i:j)×O(i:j)，只对 EV≥1.05 的比分小额下注。

> 比分适合作为主玩法的「小资金高赔率补充」，不宜做大仓位主战场。

## 六、终极硬规则（必须同时满足）

最后，给自己贴在桌子上的三条红线：

1. **从足球本质看，这一边确实更有道理**
   - 基本面、战意、伤停、对位、风格，你能讲出清晰理由；
2. **从数字上看，这一边是正期望（EV≥1.05）**
   - 不管是胜平负、让球、大小球还是比分，都满足 p×O>1；
3. **从盘面上看，不是所有人都在一边挤**
   - 没有出现「多家亚盘 / 欧指 + 本地盘一起极端压低一边」的全民热门堆积。

> 只要有一条不满足，就让这场留在笔记里，而不是留在下注记录里。

## 七、下注频率与资金管理（建议版）

> 这一节讲的是「下多少、下几场」的问题，和算得准不准一样重要。

### 7.1 下注频率：宁少不多

- 不因为比赛多就强行增加下注场次；
- 只对**完整走完本手册流程、真正满足条件**的场次下注。

参考上限：

- 单日 **1–3 场主战注**（胜平负 / 亚盘 / 大小球）；
- 其余可以视为「观察场」或小仓试验，不算进主策略。

### 7.2 单场下注比例：三档 + 上限

设总资金池为 B（只用于足彩，不动生活钱），可参考：

- **基础档（大部分场次）**：0.5%–1% B / 场；
- **信心高、模型 + 盘口高度统一的稀缺场次**：1%–2% B / 场；
- **硬上限**：
  - 单场绝不超过 **3% B**；
  - 单日全部下注金额之和不超过 **5%–6% B**。

这可以视为「比 1/10 Kelly 还保守」的注码策略，更适合长期稳健。

### 7.3 相关性控制：避免同逻辑赌太多场

同一轮、同一联赛、同一逻辑（例如都博客胜冷门 / 都博小球）往往结果相关性很强：

- 若该逻辑判断错误，容易一晚上集体亏损。

建议：

- 对每一类「逻辑 bucket」设总注额上限（例如所有客胜冷门类总注不超过 B 的 2–3%）；
- 这样即使某一逻辑翻车，损失也被限制在 1–2 个单场最大仓位之内。

### 7.4 确定下注前的最后自检

对每一场已经通过前述所有筛选的比赛，下注前再问自己一句：

- **「如果这场输掉，按现在的下注额，我在心态上、资金上都能轻松承受吗？」**

若答案是否定的：

- 要么减半下注；
- 要么本场只记在分析笔记里不下注。

> 总结：  
> **下注频率宁少不多，仓位宁轻不重。  
> 你不是在“每一场都猜对”，而是在“从很多场里只挑出少数长期正期望、且仓位受控”的局来下注。**

## 八、战绩记录与复盘（下一步必须补齐的环节）

> 目标：不再只看「今天红没红」，而是系统评估 **模型 + 决策** 的长期质量。

### 8.1 每一注需要记录什么

对每一笔真实下注（不含纯练习）至少记录：

- 时间与标识：日期、轮次、联赛、比赛 ID；
- 玩法与方向：胜平负 / 让球 / 大小球 / 比分 / 串关，具体买哪一边；
- 盘口与赔率：盘口线（如 -0.5、2.5 球）、买入时的赔率 / 水位；
- 模型视角：模型概率 P_model、对应 EV_model；
- 盘面视角：当时的主要盘面结论（例如「平热门，客冷」简要一句话）；
- 赛果：真实结果、赢盘 / 走水 / 输盘，实际盈亏。

建议统一写入同一套结构化记录（以后可由程序层管理和统计）。

### 8.2 如何按月 / 按联赛看战绩

至少从三个维度看：

- **按玩法**：胜平负 / 让球 / 大小球 / 比分，各自的下注次数、命中率、收益率；
- **按联赛**：每个联赛的推荐数、命中数、命中率、收益率；
- **按时间段**：最近 1 个月 / 3 个月 / 6 个月的整体表现曲线。

重点关注：

- 长期拖后腿的联赛 / 玩法（命中率和收益率都明显差）→ 降权甚至暂时禁玩；
- EV_model 很漂亮但实际收益长时间为负的板块 → 怀疑模型或执行策略有系统性偏差。

### 8.3 区分运气波动和模型问题

- 连红 / 连黑在短期内正常存在，不能只看几场结果；
- 用足够长的样本（例如 200+ 注）看：
  - 总 EV（Σ P×O 的理论期望）和实际收益是否一个量级；
  - EV 高的注整体是否赚钱，EV 边缘 / 低的注整体是否不稳定。

若发现：

- **整体 EV 为正但实际长期打平或亏损** → 检查记录是否完整、是否存在严重样本偏差；
- **某类场次 EV 明显为正但长期亏损** → 把这一类单独标记出来，视为模型问题重点排查对象。

## 九、择机与放弃：有 EV 也可以「不下注」

> 目标：从「看哪里有价值」升级到「哪些价值值得我真的下手」。

### 9.1 明知有 EV 也要放弃的场景

即便模型给出 EV≥1.05，下列场景也倾向 **只记在笔记里，不真下**：

- 典型乱局：极端伤停、战意怪、末轮练兵、杯赛前后大轮换等，本手册第一节已列；
- 盘面极端反常，但你搞不清信息来源：例如突然深盘 + 极限压水，却没有任何可验证的基本面；
- 自身体感状态很差：刚连黑、情绪化明显、想「一定要赢一场回来」时。

### 9.2 在整轮赛程中挑出「必须下」的少数场次

一轮很多场比赛中，可以这样筛：

1. 先用现有流程筛出所有 **EV≥1.05 的价值候选**；
2. 在这些里面，再根据：
   - 基本面是否极其清晰（你能说出 2–3 个扎实理由）；
   - 盘口是否没有极端热门信号；
   - 本联赛 / 本玩法的历史战绩是否稳定，
     选出 **1–3 场作为本轮「必须下的主战注」**；
3. 其余价值候选可以：
   - 只记分析，不下；
   - 或小额试验，不记入主策略统计。

## 十、进阶盘口与串关细节（展望）

> 这一节暂作方向性备注，具体计算细节可在程序层实现，再由 Agent 编排。

### 10.1 串关：用 EV 和相关性约束

- 串关中每一场，也应该满足 EV≥1.05，且逻辑不过度相关；
- 对同一类逻辑（比如都博客胜冷门 / 都博小球）设总注额上限；
- 串关仅作为在风险可控前提下「放大赔率」的工具，不当作主要盈利来源。

### 10.2 特殊盘口的期望拆分

对于 0.5/1、1/1.5 等「半球盘」，应通过比分分布把：

- 半赢 / 半输；
- 赢盘 / 走水 / 输盘；

分别拆开计算期望，再决定是否有足够 EV，而不是只凭盘感猜。

## 十一、模型实战调参套路（概览）

> 当发现某类比赛长期偏差时，如何系统地「改模型，而不是改心情」。

- 为每一类逻辑 / 联赛 / 玩法维持独立的战绩统计；
- 当某块长期偏差明显（例如某联赛大球老是高估），先暂停这块的实战下注，只做记录；
- 用最近若干百场的真实数据：
  - 回测不同的 λ 调整方案（例如整体压低总球、或下调部分球队风格）；
  - 验证不同的 DC 系数 / 权重方案；
- 只有在回测表现明显改善后，才把新参数推回实战，并从那一刻起重新开始统计新版本的战绩。

> 简单说：**先用战绩发现问题 → 在离线数据上改模型 → 验证通过后再上线**，而不是用几场连黑情绪化改规则。
