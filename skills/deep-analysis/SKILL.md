# 深度分析（deep-analysis）

本技能用于单场赛前深度分析。目标是用最新公司数据和结构化模型，给出胜平负、亚盘、大小球、泊松基准比分、盘赔约束推荐比分、信心度和串关适合度。

## 先读哪些参考

为降低耗时，默认只读本文件。遇到对应阶段再按需读取：

| 场景 | 读取 |
| --- | --- |
| 抓取失败、字段缺失、赔率公司口径不清 | `references/data-fetch-and-validation.md` |
| 需要展开完整 10 步分析、泊松、欧指、亚盘、大小球、波胆细则 | `references/analysis-method.md` |
| 需要组稿、写 memory、announce 结构化结果 | `references/output-templates.md` |
| 脚本失败、限流、已开赛、部分完成、多 worker 回传 | `references/error-handling.md` |
| 回溯旧版完整规则 | `references/full-skill-legacy.md`（仅历史参考，不作为当前执行依据；当前规则以本文件和上述 active references 为准） |

**执行原则**：能用本文件完成就不要加载长参考；只在具体步骤需要细则时读取对应 reference。

## 输入

支持四类输入：

1. 初筛流程传入的比赛列表或单场 matchId。
2. 龙王直接给比赛 ID，如 `2950979`。
3. 龙王给 titan007 分析页 URL，如 `https://zq.titan007.com/analysis/2950979cn.htm`。
4. 龙王给竞彩编号，如 `001`、`010`。编号解析由主会话优先完成；若 worker 收到的是编号，读取当日 memory 最后一个 `## 赛程抓取` section 解析为 matchId。

若编号不存在，直接告知“编号 {X} 在今日赛程中未找到”，不得进入抓取和分析。

## 强制数据入口

只使用仓库内脚本：

```bash
cd scripts/sporttery-sniper
npm run analyze -- {matchId或analysis URL} --history-window all --format openclaw-json
```

脚本输出必须满足：

- `kind = openclaw.analysis`
- 包含 `payload.match/detail/markets/context`
- `payload.match.matchId/league/kickoffTime/homeTeam/awayTeam/analysisUrl` 存在
- `markets` 中赔率、竞足、Crown 全指数可缺但必须如实标注缺口

脚本失败、JSON 不合法、关键字段缺失：本场标记“数据抓取失败”，不得编造数据。

## 开球时间检查

取得开球时间后立即比较当前时间：

- 当前时间 ≥ 开球时间：不做赛前分析，不写 memory，只通知“比赛已开始”。
- 距开球很近但未开赛：可分析，但必须提醒时间紧迫。
- sporttery-sniper 已过滤滚球；分析时仍须复核。

## 数据校验口径

必须检查这些结构化字段：

- `markets.asianCompanies` / `markets.asianHistories`
- `markets.overUnderCompanies` / `markets.overUnderHistories`
- `markets.europeCompanies` / `markets.europeHistories`
- `markets.jcOdds`
- `markets.crowFullIndex`

分析依据公司：

| 联赛类型 | 亚盘 | 大小球 | 欧指 |
| --- | --- | --- | --- |
| 欧洲赛事 | 澳彩(1)、皇冠(3)、365(8)、易胜博(12) | 同左 | 威廉(9)、365(8) |
| 德国联赛 | 澳彩(1)、皇冠(3)、365(8)、易胜博(12) | 同左 | 威廉(9)、365(8)、Interwetten(19) |
| 亚洲联赛 | 澳彩(1)、皇冠(3)、365(8)、易胜博(12)、马会(48) | 同左 | 澳彩(1)、皇冠(3)、365(8)、易胜博(12)、马会(48) |
| 其他 | 澳彩(1)、皇冠(3)、365(8)、易胜博(12) | 同左 | 威廉(9)、365(8) |

平博(47)不参与赛前分析；如脚本已返回可作为普通赔率参考，但不再为 CLV 专门抓取或存档。

## 执行顺序

1. 历史记忆召回：查看当日是否 10 分钟内分析过同场；若龙王明确“刷新/重新分析”，直接继续。
2. 运行 sporttery-sniper 获取最新上下文。
3. 校验赔率与数据完整性，标注缺失项。
4. 基本面分析：排名、主客场、近况、交锋、战意、赛程陷阱。
5. 伤停分析：关键位置、进攻/防守影响、轮换风险。
6. 泊松建模：输出泊松基准比分，最多 6 个；必要时保留与大小球同方向的泊松参考比分，但两套都不得超过 6 个。
7. 欧指分析：按联赛类型使用指定公司交叉认证，提炼首选/次选胜平负。
8. 亚盘分析：按指定公司交叉认证，判断上盘/下盘/走水风险。
9. 大小球分析：按指定公司交叉认证，给首选和次选两档盘口。
10. 合理性评估：基本面、泊松、欧指、亚盘、大小球、波胆市场是否一致。
11. 赔率变动解读：只总结关键异动，不默认展开全量时点表；龙王要求“展开过程”时再完整输出。
12. 关键点、疑点、核心矛盾、风险提示。
13. 输出综合评估与 memory 摘要。

## 比分规则

比分必须拆成两层。

### 1. 泊松基准比分

用途：数学基准和复盘校验。

- 来自泊松分布。
- 展示整体 Top 4～6：比分、概率、公平赔率；最多 6 个，禁止展开完整概率表。
- Crown/竞足波胆存在时，只作为市场赔率参考，不计算 EV。
- 若大小球方向与泊松 Top 6 明显冲突，可额外展示“与大小球同方向的泊松参考 Top 4”。这仍是泊松参考，不是最终推荐比分。

### 2. 盘赔约束推荐比分

用途：最终比分推荐。

**强制边界**：盘赔约束比分与泊松无关。不得从泊松 Top 4/Top 6 或完整泊松概率分布中取候选、排序或计算 EV。

**EV 废弃规则**：deep-analysis 不再计算、展示或复盘 EV。亚盘、大小球、比分都只输出方向判断、赔率依据、泊松概率/公平赔率（用于数学基准）和波胆市场参考；不得写“泊松 EV”“EV=概率×赔率”“💰/≈/❌ 价值分级”。

必须综合：

- 欧赔判断，如主胜/平/客胜。
- 亚让判断，如上盘/下盘/走水风险。
- 大小球判断，如大 3、小 2.5。
- Crown 全指数波胆比分全集。
- 竞足比分/波胆；有则与 Crown 并列参考，没有则只参考 Crown。

候选池来自 Crown 全指数波胆；有竞足比分/波胆时并入候选池。按欧赔、亚盘、大小球约束筛选并排序。

整数大小球盘口：

- 大 3：总进球 ≥3 可入候选；3 球标“走水”，4+ 球标“大球命中”。
- 小 3：总进球 ≤3 可入候选；3 球标“走水”，0-2 球标“小球命中”。
- 半球盘按明确大/小方向筛选。

盘赔约束输出不得带泊松概率、公平赔率或 EV。每个比分只写：比分、胜平负/亚盘/大小球是否满足、Crown 赔率、竞足赔率（如有）。

## 输出顺序

先发结论，再发过程。

结论建议控制在 3500 字以内，必须包含：

- 比赛信息：编号、matchId、联赛、主客队、开球时间、分析时间。
- 胜平负：首选 + 次选 + 即时赔率依据。
- 亚盘：方向、盘口、主客水、关键依据；不明确则观望。
- 大小球：首选 + 次选 + 预期总进球；不明确则观望。
- 泊松基准比分最多 6 个。
- 盘赔约束推荐比分 Top 3。
- 关键点、疑点、核心矛盾、风险。
- 信心度与串关适合度。

过程部分默认精简，只保留关键推导。龙王要求“完整过程/展开轨迹”时，再按 `references/analysis-method.md` 输出完整 10 步和赔率变动轨迹。

## Memory 摘要

memory 只写结构化摘要，不写完整分析全文。必须包含：

- 基本面摘要。
- 赔率摘要：欧指、亚盘、大小球方向和关键异动。
- 比分赔率依据：泊松基准比分只记录是否有 Crown/竞足参考；盘赔约束比分使用 Crown/Crown+竞足/竞足/Crown 缺失。
- 综合评估：胜平负、亚盘、大小球、泊松基准比分、盘赔约束比分、置信区间、关键点、疑点、核心矛盾、风险、信心度、串关适合度。
- 泊松比分简表：最多 6 个比分，供 post-review 做 Top 6 命中校验；不写完整 0:0～7:7 概率表。

多 worker 场景：worker 不写 memory，只通过 announce 返回结构化结果和 memory 摘要；编排方统一合并写入。

## 异常处理

- 抓取失败：重试 1 次；仍失败则结束本场，不写 memory。
- 部分赔率缺失：继续分析但标注缺口并降低相关维度权重。
- 比赛已开始：跳过，不写 memory。
- 超时或限流：跳过该场或降级输出，不能编造。
- 多场并行：只合并含完整 memory 摘要的 announce。
