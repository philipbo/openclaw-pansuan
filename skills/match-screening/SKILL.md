# 赛事初筛（match-screening）

对 match-scraper 抓取到的全量赛程，按 6 阶段流程筛选出 3-8 场值得深度分析的比赛。

## 输入

来自 match-scraper 的结构化赛程列表，或从当日 memory 文件中读取。

**触发方式**：

- **定时任务**（17:00）：处理全部赛程
- **手动触发**：按龙王指定的范围处理

## 执行流程

严格按以下 6 个阶段顺序执行。每个阶段都会缩小候选池。

### 阶段 0：预处理与标签

对每场比赛打标签：

1. **赛事类型**：联赛 / 杯赛 / 友谊赛 / 其他
2. **直接剔除**：
   - 友谊赛、表演赛
   - U21、U23、青年队赛事
   - 赛事名称中含「友谊」「热身」「邀请」的
3. **标记杯赛**：赛事名称中含「杯」「Cup」「欧冠」「欧联」「欧协联」的

输出：标记后的候选列表，记录剔除原因。

### 阶段 1：联赛层级筛选

按以下层级表分类：

**高层级（优先保留）**：
英超、西甲、德甲、意甲、法甲、欧冠、欧联杯、欧协联、
葡超、荷甲、亚冠精英、世界杯、欧洲杯、亚洲杯

**中层级（可选保留）**：
英冠、德乙、法乙、荷乙、日职联、韩K联、澳超、沙特联、亚冠联2、北欧

**低层级（仅有强理由时保留）**：
巴甲、阿甲、美职联、西乙、意乙、比甲、土超、东欧及其他未列联赛

**排除**：
友谊赛、表演赛、未知/无法识别赛事

规则：

- 保留所有高层级场次
- **中层级始终进入候选池**（不因高层级充足而排除——英冠保级战的分析价值可能远超英超中游无欲对决）
- 低层级仅在有明确战意或特殊信息时保留
- 最终由阶段 6 的综合评分决定优先级，阶段 1 不做数量淘汰

### 阶段 2：时间与信息可获性

对剩余场次逐一检查：

1. **时间过滤**：
   - 只要出现在赛程列表中的场次即视为可分析，**不按竞彩销售截止时间做剔除**（龙王自行把握购彩截止：周一至五 22:00、周六日 23:00）。
   - 如果比赛开球时间在当前时间之前（已开场/已完场）→ 剔除
   - **凌晨场可选剔除**：当候选场次足够多时，可剔除**次日 05:00～11:00 开球**的场次（典型如巴甲、阿甲、美职联的凌晨时段）；场次紧张时不剔除，保留分析价值

2. **分析时间过滤**（仅定时任务触发时执行，用户手动触发不受此限制）：
   - 计算：开球时间 - 当前时间
   - 如果 < 1 小时 → 剔除，理由「时间不足」
   - **用户手动触发分析时，不设时间下限**，即使临场 10 分钟也可以分析（临场赔率变化本身就是重要信号）

3. **信息标记**：
   - 国际比赛日后首轮 → 标记「轮换风险」
   - 3 天内有另一场比赛 → 标记「体能/轮换风险」
   - 下一场为欧冠淘汰赛/联赛争冠直接对话/杯赛决赛等大赛 → 标记「⚡ 赛程陷阱」（强队 vs 弱队时尤其重要）
   - 无近 5 场战绩数据（赛季初期/新升级队等）→ 标记「信息不足」（**不排除**，仍进入后续阶段，但在深度分析时需注意数据局限性）

### 阶段 3：杯赛、战意与特殊标记

**杯赛场次**：

- 淘汰赛 / 决赛 → 保留，标记「杯赛关键战」
- 小组赛且已出线/已淘汰 → 降权，标记「战意存疑」
- 无法判断战意 → 降权

**联赛场次**（需要对球队排名有判断）：

- 积分榜前 3 或后 3 → 标记「战意明确：争冠/保级」
- 争欧战区（4-7名附近）→ 标记「战意明确：争欧」
- 中游位置 → 标记「战意一般」

如果无法获取积分榜信息，跳过战意标记但不剔除。

**德比 / 特殊对决标记**：

- 同城德比（米兰德比、曼市德比、国家德比、鲁尔德比等）→ 标记「🔥 德比」，优先保留
- 传统宿敌 / 杯赛决赛 → 标记「🔥 焦点战」
- 争冠直接对话 / 保级生死战 → 标记「🔥 天王山」
- 德比和焦点战即使战意标记为一般，也不降权（特殊比赛自带高战意）

**球队级别标记**：

- 含超级豪门（皇马、巴萨、拜仁、曼城、利物浦、切尔西、曼联、尤文、巴黎、国米等）→ 标记「豪门局」
- 两支豪门对决 → 标记「豪门对话」，优先级最高
- 豪门 vs 弱旅且实力差距悬殊（如排名前 3 vs 排名后 3，或豪门 vs 升班马）→ 标记「深盘局」，**对比分投注有价值**（强队大比分相对可预测，泊松模型在实力差距大时准确度更高）

**赛季阶段标记**：

- 联赛第 1-5 轮 → 标记「赛季初期」（升班马常被低估，数据不足需警惕）
- 圣诞/冬歇前（12月英超等密集赛程期）→ 标记「密集赛程期」（冷门高发）
- 联赛最后 5 轮 → 标记「赛季末段」，重点关注战意分化：
  - 已确定降级/保级/夺冠的队 → 标记「战意存疑」
  - 仍在争夺排名的队 → 标记「战意拉满」

### 阶段 4：基础数据门槛

对剩余场次，评估基本面差异：

1. **实力反差**：
   - 如果能从赛事/球队名推断明显强弱差（如巴萨 vs 升班马），标记「实力反差大」
   - 双方实力接近 → **不降权**（均衡对决往往是赔率定价偏差最大的比赛，对比分投注有价值）
   - 注意：不用近 5 场胜率做硬性门槛，避免追涨杀跌

2. **主客因素**：
   - 强队主场 → 加分
   - 强队客场 → 可能有冷门价值，保留观察
   - 主客场表现差异大（主场龙/客场虫）→ 标记「主客反差大」

3. **伤停信息**（如在抓取阶段已获取）：
   - 有关键球员缺阵 → 标记「伤停影响」

### 阶段 5：盘赔一致性

**数据来源**：每场比赛的分析页（`https://zq.titan007.com/analysis/{matchId}cn.htm`）顶部「即时走势」模块，点击「让」「大」「欧」标签可查看约 4 家公司的初盘和即时赔率（亚盘、大小球、欧指）。

**操作方式**：对候选场次的分析页做**快速浏览**（只看即时走势汇总，不逐家公司抓取赔率变化详情——那是深度分析的事）。每页只需 snapshot 提取顶部盘口汇总，耗时约 10-15 秒/场。

**⚠️ 与 deep-analysis 的数据差异**：本阶段只获取 ~4 家公司的初盘和即时盘（2 个快照点），用于方向一致性打分。deep-analysis 步骤 1.5 会独立抓取 5-7 家公司的**完整赔率变动历史**（含每次变动时间戳），用于 RLM 检测、诱盘识别等深度排查。两者数据源和颗粒度完全不同，不存在复用关系——且赔率随时变化，间隔数小时后即时盘已不同。

如果能获取到即时盘口汇总：

1. 对比各公司欧赔/亚盘方向
2. 方向一致（都看好同一方）→ 标记「盘赔一致」（方向确定性高）
3. 方向分歧大 → 标记「盘赔分歧」，**加分而非降权**（市场未形成共识 = 定价偏差大 = 分析价值高）

如果场次较多（>10 场）导致逐场浏览耗时太长，可只对阶段 0-4 评分靠前的 Top 8 场做盘赔检查，其余标记「盘赔未知」。

### 阶段 6：综合排序与数量控制

对通过前 5 阶段的场次综合评分：

| 因子             | 权重 | 评分规则                                                                    |
| ---------------- | ---- | --------------------------------------------------------------------------- |
| 联赛层级         | 20%  | 高=3, 中=2, 低=1                                                            |
| 德比/焦点/天王山 | 15%  | 德比/焦点/天王山=3, 豪门局=2, 无=1                                          |
| 战意明确度       | 20%  | 明确=3, 一般=2, 存疑=1                                                      |
| 实力/数据反差    | 15%  | 大=3, 中=2, 小=1                                                            |
| 分析价值信号     | 20%  | 盘赔分歧=3, 有风险标记（赛程陷阱/伤停等）=2, 盘赔未知=2, 盘赔一致且无标记=1 |
| 比分可预测性     | 10%  | 深盘局/强弱分明=3, 一般=2, 混沌局=1                                         |

按总分降序排列，取 Top N 场：

- 推荐取前 3-5 场
- 最多不超过 8 场
- 如果分数最高的不足 3 场，如实报告「今日高质量场次不足」

## 输出格式

```
## 初筛结果

筛选时间：{时间}
候选总数：{原始场次数} → 通过初筛：{N} 场

### 推荐深度分析场次

1. ⭐⭐⭐ [英超] 阿森纳 vs 曼城 | 20:30 | ID: 2950977
   🔗 https://zq.titan007.com/analysis/2950977cn.htm
   标签：高层级 | 🔥 天王山 | 盘赔一致
   📌 推荐原因：积分榜1、2名直接对话，阿森纳主场近10轮8胜，曼城客场近5轮3负。欧赔全线压低主胜，亚盘主让半一高水，机构明确看好主队。争冠直接对话战意拉满，比分可预测性高。
   ⚠️ 风险：无明显风险

2. ⭐⭐⭐ [西甲] 巴萨 vs 皇马 | 22:00 | ID: 2950981
   🔗 https://zq.titan007.com/analysis/2950981cn.htm
   标签：高层级 | 🔥 德比 | 盘赔分歧
   📌 推荐原因：国家德比实力接近，盘口仅开平半，说明机构对方向犹豫。欧赔方向分歧（365偏主胜、威廉偏平局），市场未形成共识 = 定价偏差可能大，深度分析有挖掘价值。
   ⚠️ 风险：轮换风险（周中有欧冠），需在深度分析中确认首发

3. ⭐⭐ [英超] 热刺 vs 利物浦 | 23:00 | ID: 2950985
   🔗 https://zq.titan007.com/analysis/2950985cn.htm
   标签：高层级 | 争欧 vs 争冠 | 伤停影响
   📌 推荐原因：利物浦争冠需拿分，但热刺主场战意同样强（争欧关键期）。初步看亚盘客让半球低水，进球数盘口2.5偏高值得关注。热刺有关键球员伤停，需深度分析评估影响程度。
   ⚠️ 风险：热刺核心中场伤停，可能影响攻防平衡

### 被筛除的场次摘要（折叠）
- {联赛} {主队} vs {客队}（ID: {matchId}）🔗 [分析页](https://zq.titan007.com/analysis/{matchId}cn.htm) → 筛除原因：{具体原因，如「低层级+战意存疑+盘赔无异动」}
```

**推荐原因撰写要求**：

- 不是标签堆砌，而是**一段话说清楚为什么值得花时间深度分析**
- 包含以下要素（有什么写什么，不必全有）：
  - 排名/积分态势（争什么）
  - 近期走势亮点（连胜/连败/主场龙等）
  - 盘口初步信号（阶段 5 快速浏览看到的方向/异动）
  - 分析价值点（盘赔分歧=定价偏差、深盘局=比分可预测、伤停=需评估影响等）
- 2-3 句话即可，不用写长篇

**matchId 必须在输出中明确展示**，deep-analysis 需要它来拼接分析页 URL。如果 match-scraper 未能提取到 matchId，在此标注「ID 缺失」，deep-analysis 需通过其他方式（如从赛程列表页重新获取）补全。

```

```

## 输出

将初筛结果**先展示给龙王**，让其了解今日候选全貌。然后：

- **定时任务触发**：展示后自动进入 deep-analysis，无需等待确认
- **用户手动触发**：展示后等待龙王确认（龙王可以说「分析吧」直接开始，也可以说「去掉第 3 场」「加上 XX 比赛」调整候选列表后再开始）

返回通过初筛的场次列表，供 deep-analysis 使用。
