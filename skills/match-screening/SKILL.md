# 赛事初筛（match-screening）

对 match-scraper 抓取到的全量赛程，按 6 阶段流程筛选出 3-8 场**有投注价值**、值得深度分析的比赛。只筛选**当天销售**的比赛（竞彩当日开售场次）。

## 初筛本意与分析逻辑

**投注价值的含义**：筛选有投注价值的比赛，是通过分析数据、球队状态、历史对战等因素，找到可能产生较大回报或更低风险的投注机会。投注价值本质上是**投注选项（胜平负、进球数、亚让等）相对于其真实概率是否划算**——即你对比赛结果的判断/信心与当前赔率是否匹配；匹配或赔率更优则有价值，反之则无。

**目标**：找这类有投注价值的比赛。通过「自开盘 + 各公司对比 + 基本面认证」得到关键点、疑点、核心矛盾，以及各公司的操作角度与逻辑。

**分析顺序**（对进入阶段 4 的场次，打开分析页执行）：

1. **先基本面、伤情分析**，并据此**自己先开一个欧洲指数、盘口、赔率/水位**（基于基本面得出的「合理盘」）。
2. **再看各公司的盘口、赔率/水位**：各公司是否一致、有什么分歧。
3. **用基本面的分析 + 自开盘**认证各公司：结合基本面结论与自开盘，对照各公司实际盘，判断各公司盘口、赔率/水位属于何种情况（正常/偏高/偏低，多/少等），从而得到**关键点、疑点、核心矛盾**，并理解**各公司的操作角度与逻辑**。

**数据来源**：分析页 `https://zq.titan007.com/analysis/{matchId}cn.htm`。优先使用澳彩、皇冠、365、易胜博四家；若某家缺失可用其他公司替代。品种：欧洲指数（胜平负）、实际亚盘、进球数；时点：初盘 + 即时。

## 输入

来自 match-scraper 的结构化赛程列表，或从当日 memory 文件中读取。**仅对当天销售的比赛做筛选**。

**触发方式**：

- **定时任务**（17:00）：处理全部赛程
- **手动触发**：按龙王指定的范围处理

## 执行流程

严格按以下 6 个阶段顺序执行。每个阶段都会缩小候选池。

### 阶段 0：预处理与标签

对每场比赛打标签：

1. **赛事类型**：联赛 / 杯赛 / 友谊赛 / 其他
2. **直接剔除**：
   - 友谊赛、表演赛
   - U21、U23、青年队赛事
   - 赛事名称中含「友谊」「热身」「邀请」的
3. **标记杯赛**：赛事名称中含「杯」「Cup」「欧冠」「欧联」「欧协联」「亚冠」的

输出：标记后的候选列表，记录剔除原因。

### 阶段 1：联赛层级筛选

按以下层级表分类：

**高层级（优先保留）**：
英超、西甲、德甲、意甲、法甲、欧冠、欧联杯、欧协联、
葡超、荷甲、亚冠精英、世界杯、欧洲杯、亚洲杯

**中层级（可选保留）**：
英冠、德乙、法乙、荷乙、日职联、韩K联、澳超、沙特联、亚冠联2、北欧

**低层级（仅有强理由时保留）**：
巴甲、阿甲、美职联、西乙、意乙、比甲、土超、东欧及其他未列联赛

**排除**：
友谊赛、表演赛、未知/无法识别赛事

规则：

- 保留所有高层级场次
- **中层级始终进入候选池**（不因高层级充足而排除）。中层级同样有很多具备投注价值的比赛，甚至某些场次比高层级更有价值（如英冠保级、战意更清晰、盘口分歧更大等），故本阶段**只做分类、不按层级淘汰**，最终谁入选、谁排前面由阶段 4～6 的投注价值与关键点/疑点决定。
- 低层级仅在有明确战意或特殊信息时保留
- 最终由阶段 6 的综合评分决定优先级，阶段 1 不做数量淘汰

### 阶段 2：时间与信息可获性

对剩余场次逐一检查。只筛**当天销售**的比赛；已开场/已完场必须剔除。

1. **时间过滤**：
   - 开球时间在当前时间之前（已开场/已完场）→ 剔除
   - **不按竞彩销售截止时间做剔除**（龙王自行把握购彩截止：周一至五 22:00、周六日 23:00）
   - **预过滤**（当**当天比赛总数 > 10 场**时生效）：
     - 距开球 **< 30 分钟** → 剔除（时间不足）
     - **次日 05:00 及之后**开球的比赛 → 剔除（如巴甲、阿甲、美职联等凌晨场）
   - 当天比赛 ≤ 10 场时，不做上述预过滤，保留更多候选

2. **分析时间过滤**（仅定时任务触发时执行，用户手动触发不受此限制）：
   - 当天 >10 场时已见上（<30 分钟剔除）；当天 ≤10 场时：若距开球 < 1 小时 → 剔除，理由「时间不足」
   - **用户手动触发分析时，不设时间下限**，即使临场 10 分钟也可以分析

3. **信息标记**：
   - 国际比赛日后首轮 → 标记「轮换风险」
   - 3 天内有另一场比赛 → 标记「体能/轮换风险」
   - 下一场为欧冠淘汰赛/联赛争冠直接对话/杯赛决赛等大赛 → 标记「⚡ 赛程陷阱」（强队 vs 弱队时尤其重要）
   - 无近 5 场战绩数据（赛季初期/新升级队等）→ 标记「信息不足」（**不排除**，仍进入后续阶段，但在深度分析时需注意数据局限性）

### 阶段 3：杯赛、战意与特殊标记

**杯赛场次**：

- 淘汰赛 / 决赛 → 保留，标记「杯赛关键战」
- 小组赛且已出线/已淘汰 → 降权，标记「战意存疑」
- 无法判断战意 → 降权

**联赛场次**（需要对球队排名有判断）：

- 积分榜前 3 或后 3 → 标记「战意明确：争冠/保级」
- 争欧战区（4-7名附近）→ 标记「战意明确：争欧」
- 中游位置 → 标记「战意一般」

如果无法获取积分榜信息，跳过战意标记但不剔除。

**德比 / 特殊对决标记**：

- 同城德比（米兰德比、曼市德比、国家德比、鲁尔德比等）→ 标记「🔥 德比」，优先保留
- 传统宿敌 / 杯赛决赛 → 标记「🔥 焦点战」
- 争冠直接对话 / 保级生死战 → 标记「🔥 天王山」
- 德比和焦点战即使战意标记为一般，也不降权（特殊比赛自带高战意）

**球队级别标记**：

- 含超级豪门（皇马、巴萨、拜仁、曼城、利物浦、切尔西、曼联、尤文、巴黎、国米等）→ 标记「豪门局」
- 两支豪门对决 → 标记「豪门对话」，优先级最高
- 豪门 vs 弱旅且实力差距悬殊（如排名前 3 vs 排名后 3，或豪门 vs 升班马）→ 标记「深盘局」，**对比分投注有价值**（强队大比分相对可预测，泊松模型在实力差距大时准确度更高）

**赛季阶段标记**：

- 联赛第 1-5 轮 → 标记「赛季初期」（升班马常被低估，数据不足需警惕）
- 圣诞/冬歇前（12月英超等密集赛程期）→ 标记「密集赛程期」（冷门高发）
- 联赛最后 5 轮 → 标记「赛季末段」，重点关注战意分化：
  - 已确定降级/保级/夺冠的队 → 标记「战意存疑」
  - 仍在争夺排名的队 → 标记「战意拉满」

### 阶段 4：基本面与伤情 → 自开盘

对通过阶段 0-3 的场次**全部**逐一打开分析页（`https://zq.titan007.com/analysis/{matchId}cn.htm`）执行本阶段与阶段 5，不预砍；最终在阶段 6 再按投注价值与关键点排序取 Top N。

**⚠️ 即时走势比较（仅分析页）**：阶段 4、5 依赖**分析页**上的「即时走势比较」表格（各公司欧指、亚盘、进球数 初+即时）。其他页面（赛程、赔率详情等）没有该表格，无需此步骤。若打开分析页并 snapshot 后**没有**该表格或只有标题无数据，必须先按 **TOOLS.md** 中「即时走势比较：获取数据环节最重的一步」执行 **定制 → 指数比较 → 确定** 流程，再 snapshot 提取。拿错或拿不到则后续分析无效。

**从分析页提取**：

- **基本面**：联赛积分排名（双方排名、胜率、主/客/总、近 6 场）、对赛往绩、近期战绩、未来五场
- **伤情**：阵容情况（伤停名单、位置、缺阵原因）

**本阶段要做的事**：

1. 做**基本面与伤情分析**（实力反差、主客、战意、轮换风险等），并据此**自己先开一个欧洲指数、盘口、赔率/水位**（即基于基本面得出的「合理盘」），作为后续认证各公司盘的基准。
2. 标记：实力反差大/主客反差大/伤停影响等；若强队胜赔极低（如深盘局），标记「胜平负价值有限，亚让/进球数/比分更有分析价值」。

### 阶段 5：各公司盘口对比 → 认证 → 关键点/疑点/核心矛盾

**数据**：分析页**「即时走势比较」**表格中的**欧指、亚盘、进球数**的**初盘 + 即时**（各公司：澳*、Crow*、36*、易* 等，若某家缺失可用其他公司替代）。若页面上无此表格，必须按 TOOLS.md 先执行「定制 → 指数比较 → 确定」再提取。

**本阶段要做的事**：

1. **各公司盘口、赔率/水位**：提取各公司欧指、亚盘、进球数（初盘 + 即时），看各公司是否一致、有什么分歧。
2. **用基本面的分析 + 自开盘**认证各公司：结合阶段 4 的**基本面分析结论**和**自开的欧指/盘口/赔率**，对照各公司实际盘，判断各公司的盘口、赔率/水位属于何种情况（正常 / 偏高 / 偏低，多 / 少等），从而得到**关键点、疑点、核心矛盾**。
3. 理解**各公司的操作角度与逻辑**（阻/诱、抬/压等），可先都写进输出，后继再慢慢调整表述与粒度。

通过阶段 0-3 的场次均做阶段 4+5，不做预砍；耗时若偏长可后续再调。

### 阶段 6：综合排序与数量控制

对通过前 5 阶段的场次综合评分（权重与规则可后继调整）。**排序以投注价值与关键点/疑点为核心**，联赛层级为参考因子；中层级若投注价值、关键点/疑点、盘赔分歧等突出，可排在高层级前面。

| 因子             | 权重 | 评分规则                                                                                          |
| ---------------- | ---- | ------------------------------------------------------------------------------------------------- |
| 联赛层级         | 20%  | 高=3, 中=2, 低=1（仅作参考，不主导排序）                                                          |
| 德比/焦点/天王山 | 15%  | 德比/焦点/天王山=3, 豪门局=2, 无=1                                                                |
| 战意明确度       | 20%  | 明确=3, 一般=2, 存疑=1                                                                            |
| 实力/数据反差    | 15%  | 大=3, 中=2, 小=1                                                                                  |
| 分析价值信号     | 20%  | 有关键点/疑点/核心矛盾=3, 盘赔分歧=2.5, 有风险标记=2, 盘赔未知=2, 盘赔一致且无标记=1              |
| 投注价值         | 10%  | 深盘局（亚让/进球/比分价值高）=3, 均衡局有定价偏差=2.5, 一般=2, 胜平负价值低且未标亚让/进球价值=1 |

按总分降序排列，取 Top N 场：

- 推荐取前 3-5 场
- 最多不超过 8 场
- 如果分数最高的不足 3 场，如实报告「今日高质量场次不足」

**推荐原因撰写要求**：

- 不是标签堆砌，而是**一段话说清楚为什么值得花时间深度分析**
- 包含以下要素（有什么写什么，不必全有）：
  - 排名/积分态势（争什么）
  - 近期走势亮点（连胜/连败、主场龙等）
  - **关键点/疑点/核心矛盾**（阶段 5 用自开盘认证各公司后得出的结论）
  - 各公司操作角度/逻辑（阻/诱等，可简写，后继慢慢调整）
  - 投注价值侧重（胜平负 vs 亚让/进球数/比分）
- 2-3 句话即可，不用写长篇

**matchId 必须在输出中明确展示**，deep-analysis 需要它来拼接分析页 URL。如果 match-scraper 未能提取到 matchId，在此标注「ID 缺失」，deep-analysis 需通过其他方式（如从赛程列表页重新获取）补全。

## 输出

将初筛结果**先展示给龙王**，让其了解今日候选全貌。然后：

- **定时任务触发**：展示后自动进入 deep-analysis，无需等待确认
- **用户手动触发**：展示后等待龙王确认（龙王可以说「分析吧」直接开始，也可以说「去掉第 3 场」「加上 XX 比赛」调整候选列表后再开始）

返回通过初筛的场次列表，供 deep-analysis 使用。
