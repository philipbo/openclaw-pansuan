# 推荐输出（recommendation）

逐场推送 deep-analysis 的分析结果，最后发送汇总消息（含串关组合建议）。

**核心原则**：向钱冲主玩比分串关（2 串 1 为主），推荐输出需突出比分概率排序和串关适合度。

## 输入

来自 deep-analysis 的每场比赛综合评估结果。

**流程衔接说明**：recommendation 与 deep-analysis 交替执行，而非等 deep-analysis 全部完成后才启动。具体流程：

1. deep-analysis 完成第 1 场分析 → recommendation 立即推送第 1 场
2. deep-analysis 完成第 2 场分析 → recommendation 立即推送第 2 场
3. ...依此类推...
4. 全部场次分析推送完成 → recommendation 生成汇总消息（含串关建议）并推送

## 执行步骤

### 步骤 1：筛选可推荐场次

从深度分析结果中筛选：

- **信心度为「高」或「中」的场次** → 纳入推荐
- **信心度为「低」的场次** → 不推荐，但可列为「观察场次」
- 如果所有场次信心度都低 → 如实告知「今日无高信心推荐，建议观望」

### 步骤 2：排序

按以下优先级排序：

1. 信心度（高 > 中）
2. 串关适合度（🔒 > ✅ > ⚠️）
3. 联赛层级（高 > 中）

### 步骤 3：逐场推送分析报告

**分析完一场，立即推送一场**，不等全部分析完。每场推荐包含完整分析过程 + 结论：

```
📌 [{联赛}] {主队} vs {客队}（{N}/{总推荐数}）
⏰ {开球时间}

━━━ 分析过程 ━━━

1️⃣ 基本面分析
{基本面分析内容，含均值回归评估、赛程陷阱检测、赛季阶段特征}

2️⃣ 伤情分析
{伤停情况及对球队影响}

3️⃣ 欧洲指数（胜平负）赔率分析
{表格：各公司初盘→即时盘赔率（含平博锚点）}
{意图解读 + RLM 检测结果}

4️⃣ 亚盘赔率分析
{表格：各公司盘口+水位 初盘→即时盘}
{意图解读 + RLM 检测结果}

5️⃣ 进球数赔率分析
{表格：各公司大小球盘口+水位 初盘→即时盘（含平博验证）}
{意图解读 + 平博方向验证}

6️⃣ 赔率合理性评估
{三者交叉验证 + 平博锚点校准 + 价值评估}

7️⃣ 赔率变动解读
{各公司变动时间线，谁先动、方向是否一致}

8️⃣ 关键点 / 疑点 / 矛盾
{核心判断要素、不确定项、冲突信号、RLM 信号汇总}

9️⃣ 风险提示
{可能推翻判断的因素}

🔢 比分概率建模
{泊松模型 Top 8 比分概率表}
{与竞彩比分赔率的价值偏差（如可获取）}

━━━ 结论 ━━━

🎯 推荐：{推荐内容}
📊 置信区间：主胜 {XX-XX%} / 平局 {XX-XX%} / 客胜 {XX-XX%}
⚽ 比分推荐（按概率排序）：
  1️⃣ {比分} ({概率}%) {💰有价值}
  2️⃣ {比分} ({概率}%)
  3️⃣ {比分} ({概率}%)
💪 信心度：{高/中}
🔗 串关适合度：{🔒做胆推荐 / ✅可串 / ⚠️慎串}
📊 核心理由：
  1. {最关键的理由}
  2. {第二关键}
  3. {第三关键（如有）}
⚠️ 风险：{主要风险点}
```

推荐内容的表述规范：

- 胜平负：「主胜」「平局」「客胜」
- 让球：「主队让{X}球 主胜」「主队受让{X}球 主胜」
- 大小球：「大{X.5}球」「小{X.5}球」
- 比分：「比分 X:X」
- 组合推荐（信心高时可给）：「主胜 + 大2.5球」

**组合推荐统计规则**：组合推荐按玩法拆分计数。例如「主胜 + 大2.5球」算 2 条推荐，分别计入胜平负和大小球的命中统计。但在"推荐场次"维度仍算 1 场，只要其中任一玩法命中即视为该场命中。比分推荐独立计数。

**逐场推送**：每场分析完成后立即通过 messaging 推送给向钱冲，不等所有场次分析完。这样向钱冲可以边看边决策。

### 步骤 4：生成汇总消息（含串关建议）

全部场次分析完成后，发送一条汇总消息：

```
⚽ 盘算今日汇总 | {日期} {星期X}
⏳ 今日购彩截止：{22:00/23:00}（根据星期判断）

今日共筛选 {初筛总数} 场，深度分析 {分析数} 场，推荐 {推荐数} 场。

━━━ 推荐一览 ━━━

| # | 比赛 | 推荐 | 比分首选 | 信心 | 串关 |
|---|------|------|----------|------|------|
| 1 | {主队} vs {客队} | {推荐} | {比分} | {高/中} | {🔒/✅/⚠️} |
| ... | ... | ... | ... | ... | ... |

━━━ 串关建议 ━━━

🏆 推荐 2 串 1：
  组合 A：{场次1比分} + {场次2比分}（综合赔率约 {XX}）
    理由：{为什么这两场适合串}
  组合 B：{场次1比分} + {场次3比分}（综合赔率约 {XX}）
    理由：{理由}

🎲 可选 3 串 1（风险较高）：
  组合 C：{场次1} + {场次2} + {场次3}（综合赔率约 {XX}）
    理由：{理由}

📋 观察场次（信心度低，不作为推荐）：
- [{联赛}] {主队} vs {客队} — {简要原因}

⚠️ 竞彩有风险，请在可承受范围内投注。历史战绩不代表未来表现，理性购彩，量力而行。
```

**串关组合生成规则**：

- **2 串 1**（主推）：优先用 🔒做胆场次 + ✅可串场次组合
  - 两场比赛的赛事应尽量独立（不同联赛、不同时间段更佳）
  - 避免两场都是冷门方向的组合（风险叠加太大）
  - 比分串关赔率高，可以用"1 场比分 + 1 场胜平负/让球"混合串
- **3 串 1**（可选）：仅当有 3 场以上 🔒或✅时给出
  - 3 串 1 整体命中率显著降低，必须提醒风险
- **4 串 1**：仅当向钱冲主动要求时才给出，默认不推荐
- ⚠️慎串场次不纳入串关组合

### 步骤 5：推送汇总

通过 messaging 工具将汇总消息发送给向钱冲。

如果 messaging 不可用：

- 将内容写入当日 memory
- 回复告知向钱冲推荐已生成，可在对话中查看

### 步骤 6：写入记忆

将推荐内容追加写入 `memory/{今天日期}.md`：

```
## 今日推荐

推荐时间：{时间}
推荐场次：{N} 场

### 各场推荐摘要

| 比赛 | 推荐 | 比分推荐 | 信心度 | 串关 |
| ---- | ---- | -------- | ------ | ---- |
| {主队} vs {客队} | {推荐} | {比分1} / {比分2} / {比分3} | {高/中} | {🔒/✅/⚠️} |

### 串关组合

{串关建议内容}

### CLV 追踪数据（复盘用）

| 比赛 | 推荐方向 | 推荐时赔率 | 平博即时赔率 | 比赛ID |
| ---- | -------- | ---------- | ------------ | ------ |
| {主队} vs {客队} | {推荐内容} | {推荐时对应的即时赔率} | {平博对应赔率} | {matchId} |
| ... | ... | ... | ... | ... |
```

**重要**：

- 必须记录推荐时的即时赔率和平博赔率，用于赛后 CLV 计算
- 比分推荐也记录，用于复盘比分命中率

## 特殊场景处理

### 无推荐场景

如果今日无高信心推荐：

```
⚽ 盘算今日推荐 | {日期}

今日共筛选 {N} 场，深度分析 {N} 场。
经分析，今日无高信心推荐场次，建议观望。

📋 观察场次：
- {列出信心度低的场次及原因}

💡 宁可错过，不可乱推。明天见！
```

### 用户追问单场

如果向钱冲问「{某场比赛}怎么看」：

1. 检查是否已分析过（查当日 memory）
2. 如果已分析，直接输出该场分析结果
3. 如果未分析，启动 deep-analysis 单场分析再输出

### 用户要求调整

如果向钱冲说「再多推几场」或「只看英超」：

- 回到初筛结果，按新条件调整
- 必要时对额外场次补做深度分析
- 重新生成推荐和串关建议

## 输出

逐场推送 + 汇总消息（含串关建议），已推送或已写入记忆。
