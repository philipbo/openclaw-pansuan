# 赛后复盘（post-review）

获取已完场比赛的结果，对比推荐记录，计算命中率，分析失误原因，更新长期统计。

## 输入

- 需要复盘的日期（默认：昨天）
- 当日推荐记录（从 memory 中读取）

## 执行步骤

### 步骤 1：读取推荐记录

从每日记忆中获取需要复盘的推荐：

```
memory_get memory/{复盘日期}.md
```

找到推荐部分：在 memory 文件中查找所有**二级标题**为以下之一的 section：

- `## 早波推荐`
- `## 晚波推荐（全天汇总）`
- `## 推荐（*)`（\* 为时间，如 14:25）

**取文件中最后出现的一个**作为当日最终推荐（同一天既有早波又有晚波时，晚波 section 在文件末尾，以晚波为准）。从该 section 内提取精选场次表、候补场次表、CLV 追踪数据。

**区分精选场次和候补场次**：

- 精选场次：提取联赛、主队、客队、推荐内容、信心度、比赛ID
- 候补场次：同样提取，但标记为「候补」

**复盘主体 = 精选场次**（向钱冲实际投注的场次）。候补场次也做结果核对，但单独展示作为参考（"如果选了会怎样"），不计入主要命中率/CLV 统计。

**补追待复盘场次**：检查前 3 天（{复盘日期-1} 到 {复盘日期-3}）的 memory 文件，如果其中有标记为「待复盘」的场次，一并纳入本次复盘。这些场次：

- 在复盘报告中标注为「补复盘（原推荐日期：{日期}）」
- 命中率计入原推荐日的统计（更新原日期 memory 中的复盘记录）
- CLV 和泊松排位正常计算

如果没有找到推荐记录（当日 + 待复盘均无）：

- 告知向钱冲「{日期}没有推荐记录，无需复盘」
- 结束流程

### 步骤 2：获取比赛结果

从推荐记录中取出每场比赛的比赛 ID，逐场打开分析页获取最终比分：

```
browser open https://zq.titan007.com/analysis/{matchId}cn.htm
browser wait --load networkidle --timeout-ms 10000
browser snapshot
→ 提取页面顶部的最终比分（完场后页面会显示比分）
browser tab close {targetId}
```

逐场获取，每场间隔 1-2 秒。

如果推荐记录中比赛 ID 丢失，退化到列表页方案：

```
browser navigate https://jc.titan007.com/index.aspx
browser wait --load networkidle --timeout-ms 10000
→ 如需查看昨日赛程，切换日期选择器
→ browser snapshot 或 evaluate 提取已完场比赛的比分
```

### 步骤 3：逐场核对

对每场推荐的比赛：

1. **匹配比赛**：通过主客队名在赛果列表中找到对应比赛
2. **获取比分**：记录最终比分（如 2:1）
3. **判断命中**：

| 推荐类型            | 命中判断规则            |
| ------------------- | ----------------------- |
| 主胜                | 主队进球 > 客队进球     |
| 平局                | 主队进球 = 客队进球     |
| 客胜                | 客队进球 > 主队进球     |
| 让球主胜（让X球）   | 主队进球 - X > 客队进球 |
| 让球客胜（受让X球） | 客队进球 + X > 主队进球 |
| 大X.5球             | 总进球 > X.5            |
| 小X.5球             | 总进球 < X.5            |

4. **标记结果**：命中 / 未中 / 走水（让球或大小球刚好打平时）
5. **组合推荐处理**：如果是组合推荐（如「主胜 + 大2.5球」），按玩法拆分分别判断命中，计入各玩法统计；但在"推荐场次"维度仍算 1 场，任一玩法命中即视为该场命中
6. **比分预测核对**：将实际比分与推荐的 3 个预测比分对比
   - 完全命中其中一个 → 标记 🎯 精准命中
   - 未命中但总进球数一致 → 标记「进球数正确」
   - 完全不中 → 标记「比分未中」
   - 从当日 memory 读取「泊松完整概率表」，查找实际比分在概率排序中的位置（第几位、概率多少），用于评估模型质量。如果实际比分在 Top 5 内但不在推荐的 3 个中 → 标记「泊松模型准确但选取偏差」
7. **串关核对**：从当日 memory 读取串关组合建议，逐个组合判断
   - **2 串 1**：两场推荐都命中 → 串关命中；任一未中 → 串关未中
   - **3 串 1**：三场推荐全部命中 → 串关命中；任一未中 → 串关未中
   - 比分串关：两场比分都精准命中 → 串关命中
   - 混合串关（如 1 场比分 + 1 场胜平负）：各自按对应规则判断，全部命中才算串关命中
   - 记录每个组合的预期赔率和实际结果

### 步骤 3.5：分析质量自评（CLV + 泊松模型质量）

#### CLV（收盘线价值）— 针对胜平负/让球/大小球推荐

**为什么追踪 CLV？** 命中率受随机性影响大，但如果你的推荐时赔率长期优于收盘赔率，说明你拥有真正的信息优势。CLV 是衡量分析质量比命中率更可靠的指标。

**适用范围**：仅对胜平负、让球、大小球推荐计算 CLV。**比分推荐不计算 CLV**（竞彩比分赔率没有与平博可比的收盘线）。

**操作步骤**：

1. 从记忆中读取「CLV 追踪数据」表，获取每场推荐的推荐时赔率和比赛 ID

2. 逐场获取收盘赔率（比赛开始前最后一次赔率）：

```
browser open https://vip.titan007.com/changeDetail/{odds_type}.aspx?id={matchId}&companyID={companyId}&l=0
browser wait --load networkidle --timeout-ms 10000
browser snapshot
→ 提取赔率变化列表中最后一条赛前记录（即收盘赔率，注意过滤掉滚球数据）
browser tab close {targetId}
```

其中 `{odds_type}` 根据推荐方向选择：

- 胜平负推荐 → `1x2`（**优先用平博(47)**，平博不可用时用 365/威廉）
- 让球推荐 → `handicap`（**优先用平博(47)**，平博不可用时用澳彩/皇冠）
- 大小球推荐 → `overunder`（**优先用平博(47)**，平博不可用时用澳彩/皇冠）

**平博(47)是所有玩法 CLV 计算的首选基准**（三种赔率均已抓取），因为其收盘线最接近真实概率。

3. 计算每场 CLV：

```
CLV% = (收盘赔率 / 推荐时赔率) - 1

正值 = 推荐时赔率优于收盘（你比市场更早更准）→ 正 CLV ✅
负值 = 推荐时赔率劣于收盘（市场比你更准）→ 负 CLV ❌
```

4. 如果收盘赔率无法获取（页面无数据或比赛已过久），标记为「CLV 无数据」，不计入统计

#### 泊松模型质量 — 针对比分推荐

比分推荐无法用 CLV 衡量，改用以下指标评估模型质量：

- **实际比分排位**：实际比分在泊松概率表中排第几位？
  - Top 3 = 模型优秀（推荐的比分就包含正确答案）
  - Top 5 = 模型合理（方向正确但选取偏差）
  - Top 10 = 模型一般
  - Top 10 之外 = 模型偏差大，需检查 λ 值设定
- **λ 偏差方向**：实际总进球 vs 预期总进球（λ1+λ2），判断 λ 是系统性偏高还是偏低
- 长期追踪「平均泊松排位」，理想值应在 4-6 之间

### 步骤 4：分析失误

对未命中的场次，回顾深度分析记录，找出判断失误的环节：

```
memory_get memory/{复盘日期}.md
```

找到该场的深度分析部分，对照实际结果分析：

- 哪个维度的判断与实际不符？
- 是信息不足还是判断逻辑有误？
- 是否有赛前未预见的因素（红牌、伤退、天气等）？
- RLM 信号是否被正确解读？
- 泊松模型概率排序是否合理？（实际比分在 Top 几？）

输出失误分析（1-2 句话）。

### 步骤 4.5：反事实分析

对每场推荐做"如果反着买会怎样"的回顾，帮助识别系统性偏差：

- **推荐主胜但实际客胜/平局**：如果反着买（客胜/平局），赔率是多少？盈利几何？
- **推荐大球但实际小球**：反向操作收益如何？
- **比分未中**：实际比分在泊松概率表中排第几？是否在 Top 10 中？

输出格式：

```
🔄 反事实分析：
- {比赛1}：推荐{推荐} → 实际{结果}。反向操作赔率 {X.XX}，{反思}
- {比赛2}：...
```

关注的模式：

- 如果连续多天反着买都能中 → 存在系统性方向偏差，需要在策略调优中记录
- 如果实际比分总是排在泊松 Top 3 之外 → λ 值估算可能系统性偏高或偏低

### 步骤 5：计算统计

**当日统计**：

- 推荐场次数
- 命中场次数
- 命中率 = 命中数 / 推荐数
- 当日 CLV：各场 CLV% 的平均值（正值表示整体有信息优势）

**更新累计统计**：

先读取 MEMORY.md 中的现有统计：

```
memory_get MEMORY.md
```

更新：

- 总推荐 += 当日推荐数
- 总命中 += 当日命中数
- 累计命中率 = 总命中 / 总推荐
- 各联赛命中率（按联赛累加）
- 各玩法命中率（按玩法类型累加）
- 比分精准命中数 += 当日精准命中数
- 比分进球数正确数 += 当日进球数正确数
- 累计比分精准率 = 精准命中数 / 总推荐数
- CLV 统计（平博基准，仅胜平负/让球/大小球）：正 CLV 场次数、负 CLV 场次数、平均 CLV%
- 泊松模型质量：平均泊松排位、Top 3 命中率、Top 5 命中率、λ 偏差方向
- 串关统计：推荐组合数、命中组合数、按串关类型分（2串1/3串1/比分串/混合串）
- 反事实偏差监测：是否存在连续反向命中趋势

### 步骤 6：策略复盘（当累计数据 >= 20 场时）

当累计推荐达到 20 场以上，额外做一次策略复盘：

- 哪个联赛命中率最高/最低？
- 哪个玩法命中率最高/最低？
- 高信心度推荐的命中率 vs 中信心度的命中率？
- 是否需要调整策略？（如「英冠命中率持续低于 40%，建议降低英冠场次的优先级」）

将策略复盘结论写入 MEMORY.md 的「策略调优记录」部分。

```

  - {结论1}

```

### 步骤 8：写入记忆

**每日记忆**：将复盘报告追加写入 `memory/{复盘日期}.md`：

```
## 赛后复盘

{完整复盘报告}
```

**长期记忆**：更新 `MEMORY.md`：

- 更新累计战绩数字
- 更新各联赛/玩法命中率表格
- 如有新的策略结论，追加到「策略调优记录」
- 如有新的教训，追加到「关键教训」

注意：保持 MEMORY.md 精简，不超过 200 行。如果接近上限，压缩历史策略记录，只保留最近的和最重要的。

### 步骤 9：推送

通过 messaging 工具将复盘报告推送给向钱冲。

## 特殊场景

### 比赛未全部完场

如果部分推荐比赛尚未完场（如凌晨场次）：

- 先复盘已完场的
- 标记未完场的为「待复盘」，写入当日 memory
- **补追机制**：后续每次复盘时，步骤 1 会自动检查前 3 天的「待复盘」场次并补上。超过 3 天仍未完场的（推迟/取消），按「取消」处理

### 比赛取消/推迟

- 标记为「取消」或「延期」
- 不计入命中率统计

### 首次复盘（无历史数据）

- 正常计算当日统计
- 累计统计 = 当日统计
- 跳过策略复盘（数据不足）

## 输出

复盘报告（文本格式），已推送或已写入记忆。MEMORY.md 已更新。
